# Проверка бота перед запуском у клиента

Открой бота в Telegram на своём телефоне. Пройди по списку сверху вниз.
Если что-то не так — отметь крестиком и пришли скрин.

---

## 0. Подготовка (1 минута)

- [ ] `docker compose ps` — бот и redis running
- [ ] `docker compose logs bot --tail=30` — нет ошибок в последних строках
- [ ] Если тестируешь оплату локально: `python tools/mock_click_server.py` запущен в отдельном терминале
- [ ] В Telegram: удали чат с ботом (чтобы начать с чистого /start)

---

## 1. Приветствие нового клиента

- [ ] Отправь `/start`
- [ ] Должно появиться: «💅 **Sabina Nails**. Запись на маникюр и педикюр. ... Начнём? 👇»
- [ ] Внизу должна быть reply-клавиатура: `📅 Записаться` и `📋 Мои записи`
- [ ] Текст без курсива, без странных символов

---

## 2. Запись: полный флоу

Тапай кнопку `📅 Записаться`.

- [ ] Экран «Выбери направление» — две кнопки: `💅 Маникюр` и `🦶 Педикюр`
- [ ] Тапни `💅 Маникюр` → список услуг с ценами (формат: `гель-лак · 250 000`)
- [ ] Выбери любую услугу → карточка услуги с длительностью и ценой в моноширине
- [ ] Тапни «далее/выбрать» → экран выбора мастера
- [ ] Выбери мастера → календарь дат
- [ ] Выбери дату → список времён
- [ ] Выбери время → вопрос имени
- [ ] Напиши имя (2+ букв) → вопрос телефона
- [ ] Отправь телефон (кнопкой «📱 Поделиться номером» или текстом `+998901234567`)
- [ ] Экран подтверждения: «**Проверьте данные**» + таблица `Услуга / Мастер / Когда / Длительность / К оплате` в моноширине
- [ ] Две кнопки: `✅ да, записаться` и `❌ передумал`

Тапни `✅ да, записаться`:

- [ ] Hero: «✅ **ЗАПИСЬ ПОДТВЕРЖДЕНА**. Ждём, <твоё имя>.»
- [ ] Blockquote с деталями записи
- [ ] Отдельное сообщение: «🔔 Напомним за 24 часа и за 2 часа до визита.»
- [ ] Если платежи включены — ещё одно сообщение: «💳 ссылка на оплату:» с кнопкой `💳 Оплатить`

🔴 **Если после тапа `да, записаться` бот повис дольше 3 секунд** — это регрессия по UX (раньше было 4-5 сек, починили до ~1 сек).

---

## 3. «Мои записи» — пустое состояние

> ⚠️ БД помнит записи по Telegram user_id, не по чату. Удаление чата
> историю не стирает — это правильно, клиенты часто чистят чаты.
>
> Чтобы проверить пустое состояние: либо запусти бота с **другого** Telegram-аккаунта
> (у которого записей не было), либо удали записи в БД вручную:
> ```
> docker compose exec bot sqlite3 /app/data/manicure.db "DELETE FROM appointments WHERE user_id = <твой_tg_id>;"
> ```
> После — `📋 Мои записи` должны показать пустое. Если тестируешь с основным
> аккаунтом и удалять записи не хочется — **пропусти этот пункт**, переходи к §4.

- [ ] Тапни `📋 Мои записи`
- [ ] Должно быть: «📋 **У вас пока нет записей**. Первая запись — пара касаний.»
- [ ] Одна кнопка: `→ записаться`

---

## 4. «Мои записи» — одна запись

Сделай одну запись через п.2.

- [ ] Тапни `📋 Мои записи`
- [ ] Должно быть:
  - Заголовок `📋 МОИ ЗАПИСИ`
  - Секция `БЛИЖАЙШАЯ · завтра` (или «сегодня», или «пт · 25 апр»)
  - Карточка в моноширине: `Услуга / Мастер / Когда / К оплате` с `⏳ ждёт оплаты` если не оплатил
  - Секций `ЕЩЁ ПРЕДСТОЯЩИЕ` и `ИСТОРИЯ` нет (нечем заполнить)
- [ ] Кнопка: `📋 <услуга> · <когда> · <время>` (одна, ведёт в деталь)

Тапни эту кнопку:

- [ ] Карточка детали в моноширине: `Услуга / Мастер / Дата / Время / Цена / Статус`
- [ ] Кнопка `💳 Оплатить` (если не оплатил и mock Click работает)
- [ ] Кнопка `❌ Отменить запись`
- [ ] Кнопка `← Мои записи`

🔴 **Если кнопки `💳 Оплатить` нет** — проверь логи (`docker compose logs bot | Select-String resolve_pay_url`). Нормально только если провайдер недоступен.

---

## 5. Отмена неоплаченной записи

В деталях записи тапни `❌ Отменить запись`:

- [ ] Экран: «**Точно отменить запись?** 💅 <услуга> 📅 <дата·время>»
- [ ] **БЕЗ** предупреждения про возврат (запись не оплачена)
- [ ] Кнопки `❌ Да, отменить` / `← Нет`

Тапни `❌ Да, отменить`:

- [ ] Экран: «**Что случилось?**» + список причин
- [ ] Выбери любую причину

- [ ] Финал: «❌ **Запись отменена**. Если передумаешь — мы рядом.»
- [ ] Кнопка `📅 Записаться снова`
- [ ] **НЕТ** блока про возврат денег (не было оплаты)

Проверь у админа (в своём втором Telegram-аккаунте или у Сабины):
- [ ] Пришло уведомление `⚠️ Клиент отменил запись` с данными и причиной
- [ ] **БЕЗ** метки `💰 БЫЛА ОПЛАЧЕНА`

---

## 6. Оплата (локально через mock)

Сделай новую запись. В pay-сообщении тапни `💳 Оплатить`:

- [ ] Откроется браузер на http://localhost:8444/mock-click/pay?...
- [ ] Страница с кнопкой «Оплатить»

Тапни её:

- [ ] Вернись в Telegram
- [ ] Сообщение «💳 ссылка на оплату» **должно исчезнуть** из чата
- [ ] Должно прийти: «✅ **оплата получена.** жду тебя.»

Проверь у админа:
- [ ] Пришло `💰 **Оплата прошла**` с данными записи

Открой `📋 Мои записи`:

- [ ] В карточке ближайшей: `К оплате: 250 000 сум  💰 оплачено`
- [ ] Тапни `📋 <услуга>` → в детали НЕТ кнопки `💳 Оплатить`
- [ ] В статусе: `💰` или «ждёт» в зависимости от словаря

---

## 7. Отмена оплаченной записи

Оплати запись по п.6. Тапни `❌ Отменить запись` в деталях:

- [ ] Экран подтверждения содержит: «⚠️ **Запись оплачена. Деньги вернёт салон — свяжись для возврата.**»
- [ ] Тапни `❌ Да, отменить` → выбери причину

Финал:
- [ ] «❌ **Запись отменена**»
- [ ] Блок «💰 Оплата подлежит возврату»
- [ ] Если админ задал контакт в настройках → «📞 по вопросу возврата оплаты — @handle»
- [ ] Если не задал → «📞 свяжись с салоном по вопросу возврата оплаты»

Проверь у админа:
- [ ] В заголовке уведомления метка `💰 БЫЛА ОПЛАЧЕНА`
- [ ] В конце: «💸 Сделай возврат вручную в дашборде click.»

🔴 **Это критический момент**: если метки `БЫЛА ОПЛАЧЕНА` нет — админ не сделает refund и у клиента зависнут деньги.

---

## 8. Несколько записей — группировка

Создай 3-4 записи на разные даты. Одну отмени, одну сделай завтрашней.

- [ ] В `📋 Мои записи` видны три секции: `БЛИЖАЙШАЯ` → `ЕЩЁ ПРЕДСТОЯЩИЕ` → `ИСТОРИЯ`
- [ ] Ближайшая — с относительной датой («завтра» / «пт · 25 апр»)
- [ ] В «Ещё предстоящие» — предстоящие кроме ближайшей, статус оплаты (⏳/💰)
- [ ] В «История» — отменённая с `❌`, выполненная с `✅`
- [ ] Одна кнопка `📋` ведёт в ближайшую
- [ ] Если есть completed — кнопка `🔄 Повторить: <услуга>`

Тапни `🔄 Повторить`:
- [ ] Откроется флоу с предзаполненной услугой (меню мастера)

---

## 9. Ручная пометка «оплачено» админом

В админ-чате открой любую **неоплаченную** запись:
- [ ] Видна кнопка `💰 Пометить оплачено`

Тапни её:
- [ ] Alert: `✅ Помечено оплаченной`
- [ ] В карточке pill меняется на `💰 Оплачено`
- [ ] В чате клиента сообщение `💳 ссылка на оплату` **исчезло** (если не истекло 48 часов)

---

## 10. Reply-кнопки

В обычном сообщении (не в FSM):
- [ ] Тапни `📅 Записаться` → стартует флоу выбора категории
- [ ] Тапни `📋 Мои записи` → открывается список

Когда в FSM (например, выбираешь время):
- [ ] Тапни reply-кнопку `📋 Мои записи` → FSM сбрасывается, показывается история

---

## 11. Напоминания (проверка расписания)

Это невозможно проверить мгновенно. Но:
- [ ] Создай запись на **сегодня** через 2-3 часа
- [ ] Подожди, пока пройдёт время `(запись - 2 часа)`
- [ ] Должно прийти: «⏰ **ЧЕРЕЗ 2 ЧАСА** ...» с кнопками confirm/cancel
- [ ] Тапни confirm → «✅ ждём тебя.» + карточка
- [ ] Тапни cancel из напоминания → «❌ Запись отменена»

🔴 Если за 2 часа до записи ничего не пришло — scheduler лежит или запись не scheduled.

---

## 12. Ошибки / Edge cases

- [ ] Попробуй оплатить **уже оплаченную** запись (тапни старую кнопку `💳 Оплатить` в chat, если сохранилась) → mock покажет payment, но бот не должен пометить второй раз (в логе `duplicate`), Click получит error=-4
- [ ] Попробуй отменить через reply-кнопку **уже отменённую** запись (через «мои записи») → кнопки отмены не должно быть
- [ ] Набери в чат бота мусор типа `asdfg` → не должен висеть (fallback возвращает в меню)
- [ ] Переключи язык через `/language` → все клиентские тексты на узбекском
