# Лицензирование

Бот работает по оффлайн-верифицируемому лицензионному ключу. Нет онлайн-активации,
нет обфускации, нет ломки кода. Это **speed bump против пиратства**, не DRM.

> ⚠ **Enforcement пока выключен.** Код лицензий (подписи, верификация, grace, heartbeat)
> работает и логирует режим, но middleware-гейт в `bot.py` закомментирован. Причина:
> первые 20 клиентов — нетехнические узбекские салоны, пиратить некому, а ложное
> срабатывание вырубит работающий салон в рабочий день. Когда дойдёшь до ~20
> клиентов или поймаешь первую попытку пиратства — расскомментируй 3 строки
> в `bot.py` рядом с пометкой `# TODO` и enforcement оживёт. Всё остальное
> — выпуск ключей, установка, heartbeat, `/status` — работает уже сейчас, привыкай к флоу.

## Модель

- Каждая лицензия — подписанный Ed25519-ключом JSON-пейлоад: `tenant_slug`,
  `customer_name`, `license_id`, `issued_at`, `expires_at`.
- Подпись проверяется публичным ключом, **вмёрзнутым в `utils/license.py`** (не в файле рядом — чтобы случайно не удалили).
- Приватный ключ хранится у автора локально, **никогда не коммитится**.

## Одноразовая настройка (делается автором раз, в начале продаж)

```bash
python tools/generate_keys.py
```

Это создаст `license_private_key.pem` (сохраняй в менеджере паролей / защищённом бэкапе)
и выведет в stdout публичный ключ в формате PEM.

Скопируй весь блок PEM (включая строки `-----BEGIN PUBLIC KEY-----` и `-----END PUBLIC KEY-----`)
в переменную `PUBLIC_KEY_PEM` в `utils/license.py`, заменив плейсхолдер. Закоммить:

```bash
git commit -am "chore: set production license public key"
```

> ⚠ Если потеряешь приватный ключ — все выпущенные лицензии станут невосстановимыми.
> Если перевыпустишь ключи — все существующие клиенты мгновенно попадут в restricted
> при следующем рестарте. Делай **один раз**.

## Продажа салону (каждый раз при новой лицензии)

1. Выпустить ключ локально:
   ```bash
   python tools/issue_license.py nails-chilanzar "Салон Сабина" 12
   ```
   `12` = срок в месяцах. В stdout — строка лицензии; в stderr — саммари для твоего лога продаж.

2. Положи эту строку в `.env` клиента (через `install.sh` — мастер спросит, или вручную):
   ```
   LICENSE_KEY=<строка>
   TENANT_SLUG=nails-chilanzar   # обязан совпадать с tenant_slug в ключе!
   LICENSE_CONTACT=@sabina_nails_author
   ```

3. `docker compose up -d --build`. В логах должно появиться:
   ```
   License mode=ok
   ```

## Продление

Выписать новую лицензию на тот же `tenant_slug` с нужным сроком → заменить
`LICENSE_KEY` в `.env` клиента → `docker compose restart bot`.

## Отзыв

Просто не выписывать новую. За 7 дней до истечения бот начнёт предупреждать
админа в Telegram; через 7 дней после — перейдёт в restricted.

## Режимы работы

| Режим | Когда | Что видит клиент |
|---|---|---|
| `dev` | Публичный ключ = плейсхолдер (локальная разработка) | Всё работает, warning в логах |
| `ok` | Валидная подпись, не истекла | Всё работает |
| `grace` | Истекла ≤ 7 дней назад | Всё работает; админ получает TG-предупреждение на старте |
| `restricted` | Нет ключа / неверная подпись / tenant mismatch / истекла > 7 дней | `/start` отвечает «лицензия истекла, обратитесь к X»; остальное игнорируется |

## Heartbeat

Если задан `HEARTBEAT_URL`, бот раз в 24ч POST-ит туда `{tenant_slug, license_id, version, last_seen}`.
Провал запроса **не** блокирует бот — это телеметрия для автора, не контроль доступа.

Рекомендуемый хостинг endpoint'а: Cloudflare Worker или Fly.io (бесплатно на этом масштабе).

## Проверка всей цепочки (для автора)

```bash
# 1. Сгенерировать ключи (ОДИН раз). Вклеить публичный PEM в utils/license.py.
python tools/generate_keys.py

# 2. Выписать короткую лицензию (истекает через 60 сек — для проверки grace/restricted).
python tools/issue_license.py my-tenant "Test Salon" 1 \
    --expires-at "$(python -c 'from datetime import datetime, timezone, timedelta; print((datetime.now(timezone.utc) + timedelta(seconds=60)).isoformat())')"

# 3. Положить в .env, запустить бот → mode=ok.
# 4. Подождать 70 сек, рестарт → mode=grace, админу пуш.
# 5. Выписать уже истёкшую лицензию (за пределами grace):
python tools/issue_license.py my-tenant "Test" 1 --expires-at "2020-01-01T00:00:00+00:00"
# 6. Рестарт → mode=restricted, бот отвечает только на /start.
```

## Отладка

- `/status` показывает `License mode=...` в логах при старте — смотри `docker compose logs bot | grep License`.
- Если бот стартует но клиенты ничего не могут сделать — проверь что `TENANT_SLUG` в
  `.env` совпадает с `tenant_slug` в выписанном ключе. Самая частая причина.
