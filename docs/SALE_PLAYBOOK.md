# Sale Playbook — продажа одного салона под ключ

> **Для автора, не для клиента.** Это твой чек-лист на каждую продажу.
> Клиентский документ — `docs/INSTALL.md`.
>
> Цель — ничего не забыть в 22:00 когда ставишь 15-му салону и голова кипит.
> Проходи сверху вниз, помечай галочкой, храни журнал продаж.

---

## Этап 1 — Pre-sale: разговор с клиентом

Узнай **до** того как что-либо делать:

- [ ] **Контакт клиента** — Telegram @handle (чтобы писать напрямую).
- [ ] **Название салона** — для маркировки (не критично для кода).
- [ ] **Город** — нужен для `TIMEZONE`. Ташкент = `Asia/Tashkent`, Москва = `Europe/Moscow`, и т.д. (IANA-формат).
- [ ] **Есть ли VPS у клиента?** Если нет — ты арендуешь, включая в стоимость.
- [ ] **Telegram user_id админа** — того человека, кто будет управлять ботом.
  Как узнать: клиент пишет боту [@userinfobot](https://t.me/userinfobot) → получает числовой ID → присылает тебе.
- [ ] **Сколько мастеров** в салоне — информативно, влияет на то, как рассказать про кабинет мастера (v.3).
- [ ] **Согласована цена и срок поддержки.** Запиши словами что обещано.

Если клиент из Узбекистана — не забудь рассказать, что бот поддерживает `uz_UZ`
в будущем, но сейчас только русский (в `FUTURE.md` триггер на первого неруса).

## Этап 2 — Prep: подготовка на своей машине

### 2.1. Придумай `tenant_slug`

Правила: `[a-z0-9-]+`, без пробелов, без апострофов, без заглавных.

Примеры: `nails-chilanzar`, `beauty-mirzo`, `sabina-yunusabad`.

Не используй повторные `slug` — каждый должен быть уникален.

### 2.2. Создай Telegram-бота

Автор салона может сам, но чаще удобнее сделать самому и передать.

- [ ] Открой [@BotFather](https://t.me/BotFather) → `/newbot`.
- [ ] Имя бота: читаемое, типа «Салон Сабина — запись».
- [ ] Username: `sabina_nails_bot` (или аналог, должен кончаться на `bot`).
- [ ] Скопируй токен из ответа BotFather. **Это пароль от бота**, не шли через SMS / в переписку клиента.
- [ ] Опционально: `/setdescription`, `/setabouttext`, `/setuserpic` через BotFather — сделает бот человечнее.
- [ ] Опционально: `/setcommands` → `start - начать запись` (одна строка).

### 2.3. Выпиши 5-летнюю лицензию

```bash
cd ~/PycharmProjects/manicure
python tools/issue_license.py <tenant_slug> "<Название Салона>" 60
```

**60 месяцев = 5 лет.** Для первых 30 клиентов — всегда 60.

- [ ] Скопируй строку-ключ из stdout (длинная, `payload.sig`).
- [ ] Скопируй саммари из stderr — оно пойдёт в журнал продаж.

### 2.4. Запиши в свой журнал продаж

Веди отдельный файл у себя, **НЕ в репозитории** (там приватные данные клиентов).

Пример: `~/manicure-sales.md` (gitignored репозиторий или просто локальный файл).

Шаблон записи:
```
## 2026-04-21 · Салон Сабина

- tenant_slug:   nails-chilanzar
- customer:      Сабина Юнусабад
- contact:       @sabina_handle, +998 90 xxx-xx-xx
- tg_admin_id:   7082498953
- license_id:    <uuid из stderr>
- issued_at:     2026-04-21
- expires_at:    2031-04-20
- vps:           DigitalOcean NY3, IP 143.244.x.x
- bot_username:  @sabina_nails_bot
- bot_token:     stored in 1Password, note «Сабина»
- amount:        $XXX
- support_until: 2026-05-21 (1 месяц)
```

## Этап 3 — VPS: если у клиента нет

Пропусти, если у клиента свой VPS (просто попроси SSH-доступ).

### 3.1. Провайдер

| Провайдер | $ /мес | Где | Особенность |
|---|---|---|---|
| UzCloud (uzcloud.uz) | ~$5 | Ташкент | Локально, оплата в сумах |
| Hetzner Cloud | €4-5 | Германия | Дешёвый, надёжный |
| DigitalOcean | $6 | по миру | Классика, чуть дороже |

Минимум: **1 vCPU, 1 GB RAM, 20 GB SSD, Ubuntu 22.04**.

Для УЗ-рынка лучше UzCloud — клиент не думает про валюту, локация ближе.

### 3.2. Первичная настройка VPS

SSH под `root` (первый вход от провайдера):

```bash
# обновление
apt update && apt upgrade -y

# создать non-root пользователя (опционально, но правильно)
adduser salon && usermod -aG sudo salon

# docker одной строкой
curl -fsSL https://get.docker.com | sh
apt install -y docker-compose-plugin

# убедись что docker работает
docker run hello-world
```

## Этап 4 — Install: развёртка бота

### 4.1. Клонировать проект

```bash
cd /opt
git clone https://github.com/RiobVO/manicure.git
cd manicure
```

### 4.2. Запустить установщик

```bash
./install.sh <tenant_slug> <admin_id>
```

Интерактивно ответит на вопросы — держи под рукой:

- [ ] **BOT_TOKEN** (ввод скрыт) — из BotFather (шаг 2.2).
- [ ] **BACKUP_CHAT_ID** — твой приватный TG-канал для бэкапов всех клиентов. Создай один раз, используй везде. Как узнать ID канала: [@getidsbot](https://t.me/getidsbot). Начинается с `-100...`.
- [ ] **ERROR_CHAT_ID** — тот же канал или отдельный для ошибок. Можно тот же.
- [ ] **LICENSE_KEY** — длинная строка из шага 2.3.
- [ ] **HEARTBEAT_URL** — пока можно пропустить (пустой Enter). Триггер поднять свой endpoint — 3+ клиента.
- [ ] **LICENSE_CONTACT** — твой @handle, например `@plssog`. Будет в сообщениях «лицензия истекла, обратитесь к X».

Установщик сам:
- соберёт docker-образ,
- запустит redis, bot, autoheal,
- подождёт пока бот стартует,
- покажет последние 20 строк логов.

### 4.3. Проверить что всё поднялось

```bash
docker compose ps
# должно быть 3 контейнера: bot, redis, autoheal — все Up/healthy
docker compose logs bot --tail 30
# в логах ожидаемо: "License mode=ok ... Бот запущен"
```

### 4.4. Добавить бот в BACKUP_CHAT_ID и ERROR_CHAT_ID

Иначе он не сможет туда слать. В твоём канале:
- Управление каналом → Администраторы → Добавить администратора → найти бота по username → выдать права (минимум: отправка сообщений).

## Этап 5 — Smoke-тест (не пропускай!)

От своего аккаунта админа (тот user_id, что в `.env`):

- [ ] `/start` → появляется админ-панель с reply-клавиатурой.
- [ ] `/status` → видна строка `License mode=ok`, размер БД, uptime, Redis=✅.
- [ ] Добавь одну услугу через «💅 Услуги» → «➕ Добавить услугу».
- [ ] Добавь мастера через «👨‍🎨 Мастера» → «➕ Добавить мастера» (если их >1 — иначе дефолтный «Мастер» уже есть).
- [ ] Проверь «⚙️ Настройки» → «📅 График по дням» — дни недели настроены.

От другого аккаунта (свой клиентский, не админский):

- [ ] `/start` → приветствие нового клиента, кнопка «записаться».
- [ ] Пройди флоу до подтверждения записи.
- [ ] Админ-аккаунт должен получить пуш «🔔 Новая запись».

Если всё прошло — бот боевой. Если что-то упало — `docker compose logs bot` и дебаг.

## Этап 6 — Handover: передача клиенту

Пришли клиенту одним сообщением:

```
Бот готов: @sabina_nails_bot

Что сделать:
1. Напиши боту /start со своего телефона (Telegram user_id <admin_id>).
2. Откроется админ-панель. Настрой услуги, мастеров, расписание.
3. Добавь ссылку на бота в Instagram / WhatsApp / визитки / QR на ресепшн.

Инструкция по админке: https://t.me/<твой_канал_с_мануалом>
Поддержка: @plssog — пиши сразу, не накапливай.
```

- [ ] Отправил инструкцию.
- [ ] Клиент написал `/start`, получил панель админа.
- [ ] Клиент добавил хотя бы одну тестовую услугу.

## Этап 7 — Post-sale: что помнить

### 7.1. Первый месяц поддержки

Договорённость: первый месяц баги и вопросы — бесплатно. Дальше — платная поддержка либо на усмотрение.

Пиши в журнал что спрашивали чаще всего — это материал для FAQ.

### 7.2. Алерт об истечении лицензии

**Автоматически**: за 60 дней до `expires_at` бот шлёт тебе пуш в `ERROR_CHAT_ID` с именем клиента и готовой командой продления.

Для первых 30 с 5-летними лицензиями этот алерт сработает только через ~4.8 года — пока не думай.

### 7.3. Продление

Когда получил алерт:

```bash
python tools/issue_license.py <tenant_slug> "<Название>" 60
```

Выдаёт новый ключ → отправь клиенту инструкцию:

1. Откройте `.env` на сервере (`sudo nano /opt/manicure/.env`).
2. Замените `LICENSE_KEY=...` на новую строку.
3. `docker compose restart bot`.

Или сделай сам, если у тебя есть SSH-доступ.

### 7.4. Обновления кода (все клиенты сразу)

Когда пушишь новую версию в `main`:

```bash
# на сервере каждого клиента:
cd /opt/manicure && ./scripts/update.sh
```

Даунтайм ~30 сек. Можно автоматизировать позже через централизованный update-скрипт.

## Этап 8 — Авария / клиент ушёл

### 8.1. VPS умер

1. Возьми свежий бэкап из твоего TG-канала (ищи caption `[<tenant_slug>] backup ...`).
2. Подними новый VPS (Этап 3).
3. Восстанови через `docs/RESTORE.md`:
   ```bash
   git clone https://github.com/RiobVO/manicure.git /opt/manicure
   cd /opt/manicure
   ./install.sh <тот же tenant_slug> <тот же admin_id>
   # после установки:
   ./scripts/restore.sh ~/downloaded-backup.db
   ```

### 8.2. Клиент уходит

Запусти на его VPS:

```bash
cd /opt/manicure && ./scripts/uninstall.sh
```

- Заархивирует `data/`, `backups/`, `.env` в `/root/manicure-<slug>-<дата>.tar.gz`.
- Снесёт контейнеры и volumes.
- Архив — клиенту.

Запиши в журнал: «ушёл 2027-XX-XX, архив передан».

